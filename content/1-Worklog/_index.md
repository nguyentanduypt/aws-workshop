**On this page**, I summarize my 12-week internship journey, including the AWS services I learned, the technologies I applied, and the practical tasks I completed while developing and deploying a full-stack E-commerce application on AWS.

**Week 1:** [Getting familiar with FCAJ, setting up AWS accounts, IAM, and MFA](1.1-week1/)

**Week 2:** [Learning Amazon VPC, Security Groups, and Amazon EC2 fundamentals](1.2-week2/)

**Week 3:** [Learning Amazon S3, practicing advanced EC2 features, and managing AWS resources](1.3-week3/)

**Week 4:** [Learning Amazon EBS, Snapshots, Elastic IP, Amazon RDS, and AWS Backup](1.4-week4/)

**Week 5:** [Designing the database, initializing the full-stack project (Spring Boot, React, and PostgreSQL), and developing core APIs](1.5-week5/)

**Week 6:** [Learning Application Load Balancer (ALB), Network Load Balancer (NLB), Target Group Health Checks, and Auto Scaling Group (ASG)](1.6-week6/)

**Week 7:** [Learning Amazon CloudWatch for system monitoring and writing the worklog](1.7-week7/)

**Week 8:** [Installing Redis locally with Docker and learning Amazon ElastiCache](1.8-week8/)

**Week 9:** [Designing the AWS architecture diagram and integrating Redis into the E-commerce project](1.9-week9/)

**Week 10:** [Creating Dockerfiles for Spring Boot and React, fixing issues, and completing the project](1.10-week10/)

**Week 11:** [Containerizing the entire system, configuring CI/CD, and deploying the complete application to AWS](1.11-week11/)

**Week 12:** [Writing the internship report, preparing the graduation project report using Hugo templates, learning AWS KMS and Kiro, and finalizing the worklog](1.12-week12/)
